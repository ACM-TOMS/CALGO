SCASY - Parallel ScaLAPACK-style High Performance Library for Sylvester-Type Matrix Equations

Robert Granat and Bo Kågström

Introduction

Matrix equations arise in a number of applications in science and engineering, for example in eigenvalue problems, condition estimation and control system design and analysis. SCASY is a HPC parallel ScaLAPACK-style library for solving 44 sign and transpose variants of eight common general Sylvester-type matrix equations.

SCASY implements Bartels-Stewart's-like methods for the considered matrix equations which include four major steps:

  1. Reduction to triangular forms
  2. Update of the right hand side matrix or matrices with respect to the reduction(s) in step 1
  3. Computing the solution of the triangular matrix equation
  4. Transforming the solution to the triangular problem back to the original coordinate system

The reductions in step 1 and the transformations in step 2 and 4 are performed by orthogonal transformations only. Step 1 is carried out by using existing software in ScaLAPACK or new preliminary developments of fast and reliable algorithms for unsymmetric eigenvalue problems (parallel variants of the unsymmetric QR and QZ algorithms). Steps 2 and 4 are parallel matrix multiplications, which are implemented using PBLAS. In the 3rd step, specially designed parallel wave-front algorithms are employed that compute the solution to the triangular problems in a fixed number of operations; the major contributions from SCASY belong to this kind of algorithms and software.

For example, SCASY solves for the matrix X in eight sign and transpose variants of the general standard continuous-time Sylvester equation

op(A)X ± Xop(B) = scale C,

where 'op()' denotes a possible transpose and scale is a scalar to prevent overflow in the right hand side. Moreover, for all Lyapunov equations, the right hand side and the solution matrices may be symmetric.

We differentiate between one-sided and two-sided matrix equations, i.e, matrix equations where the solution is multiplied by another matrix from one or two sides. We also differentiate between standard and generalized matrix equations. In particular, in Table 1, the matrix equations SYCT, LYCT and GCSY are one-sided, while the remaining five are two-sided matrix equations. In addition, the matrix equations GCSY, GSYL, GLYCT and GLYDT are generalized variants of the standard matrix equations SYCT, LYCT, SYDT and LYDT.

With the solver routines in SCASY, it is possible to work with submatrices of distributed global matrices, as well. Below we denote the M-by-N submatrix of the global distributed matrix A starting at row and column indicies (IA,JA) by sub(A), where sub(A)=A(IA:IA+M-1,JA:JA+N-1).

In general, the solver PGE[ACRO]D solves for the matrix equation with the corresponding acronym ACRO. The routines calls eight corresponding triangular solvers PTR[ACRO]D which implements the parallel wave-front algorithms decribed above. However, it is not recommended that the user calls these triangular solves directly, since the major part of the error checking is performed by the general solver. The reduction and transformation steps can also be omitted by setting the in parameters to the general routine properly.

Condition estimators

SCASY provides condition estimators for the triangular counterparts of the Sylvester-type matrix equations in Table 1. The reduction to (quasi-)triangular form of the actual equation can be conducted by invoking the standard solver routine with the inargument 'JOB' to reduction mode. All condition estimators build on the ScaLAPACK condition estimator PDLACON. In analogy with the naming scheme for the solver routines, the condition estimators go under the names P[ACRO]CON, where ACRO is replaced by the corresponding matrix equation acronym.

Matrix generators

SCASY includes two generator routines for generating matrices and matrix pairs with specified standard and generalized eigenvalues. These can be utilized to construct test examples with varying conditioning in the sense of sep[ACRO].

New versions of the reduction routines

Future releases of SCASY will include new complete implementations of the unsymmetric multishift QR and QZ algorithms with advanced deflations techniques which are one to two orders of magnitude faster than the existing ScaLAPACK implementation. However, preliminary implementations of the new QR algorithm can be tested in the reduction step to triangular form of the standard equations SYCT, LYCT, SYDT and LYDT, see above, by a specific pre-processing options (see the README file and the Users' Guide in the software package for details).  The routines connected to the QZ algorithm are in the current release empty wrappers which perform no computations, but specify the assumed interface of the future production code.

Prerequisites

To use SCASY on your system you need the following.

  • A Fortran 90/95 compiler. We have tested the software with Portland Groups and the Pathscale Compiler Suites, as well as the NAG Fortran Compiler, IBMs XLFortran Compiler, and gfortran. You should have no problem with compiling the software as long as you follow the instructions in the README file and use the proper compilation flags.
  • A good implementation of the BLAS, e.g., ATLAS or the GOTO-BLAS.
  • The LAPACK library.
  • The RECSY library (which also depends on a few routines from SLICOT).
  • The ScaLAPACK library, including PBLAS and BLACS.

Installation instructions

There is currently no installation routine. However, the built library should be moved to a proper directory, e.g., /usr/local/lib.

After unpacking, the library is built as follows. Move into the SCASY directory. Then, edit the Make_include file to fit for your system regarding compiler, linker, flags, library paths, pre-processing options etc. (Make_include templates are provided for the Portland Group and the Pathscale Compiler Suites, the GNU Fortran (gfortran) compiler and the NAG Fortran Compiler.) Then try

host:~> make all

to build the library with a test program. To build only the library try

host:~> make libscasy
