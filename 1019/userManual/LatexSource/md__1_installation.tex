Prebuild binary packages (Ubuntu 18.\+04 and Ubuntu 20.\+04) are available under \href{https://github.com/NLAFET/StarNEig/releases}{\texttt{ releases}} and can be installed with the following command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\$ sudo apt install ./StarNEig-\/v0.xx.yy-\/ubuntu-\/vv.uu.deb}
\end{DoxyCode}


The binary packages rely on mainstream Star\+PU packages and do not necessary provide full functionality.

For full functionality, it is recommended that Star\+N\+Eig (and Star\+PU) are compiled from the source code, see below and/or the Star\+N\+Eig User\textquotesingle{}s Guide. Please consider using one of the tested \href{https://github.com/NLAFET/StarNEig/releases}{\texttt{ release}} versions.

\item M\+PI (optional)
\item C\+U\+DA (optional)
\item Sca\+L\+A\+P\+A\+CK + B\+L\+A\+CS (optional)

Test program and example code dependencies\+:


\begin{DoxyItemize}
\item pkg-\/config
\item G\+NU Scientific Library (optional)
\item M\+A\+G\+MA (optional)
\end{DoxyItemize}

\begin{DoxyAttention}{Attention}
Star\+N\+Eig supports Open\+B\+L\+AS, M\+KL and Goto\+B\+L\+AS. For optimal performance, a multi-\/threaded variant of one of the listed B\+L\+AS libraries must be provided. Star\+N\+Eig will automatically set the B\+L\+AS library to single-\/threaded more when necessary. If a different B\+L\+AS library is provided, then the user is responsible for setting the B\+L\+AS library to {\itshape single-\/threaded} mode. However, the use of a non-\/supported B\+L\+AS library can still impact the performance negatively.
\end{DoxyAttention}

\doxysection*{Configuration}

It is recommended that a user builds the library in a separate build directory\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\$ cd path\_to\_the\_top\_directory/}
\DoxyCodeLine{\$ mkdir build}
\DoxyCodeLine{\$ cd build}
\end{DoxyCode}


The library is configured with the {\ttfamily cmake} command. In most cases, it is not necessary to give this command any additional arguments\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\$ cmake ../}
\DoxyCodeLine{...}
\DoxyCodeLine{-\/-\/ Configuring done}
\DoxyCodeLine{-\/-\/ Generating done}
\DoxyCodeLine{-\/-\/ Build files have been written to: /.../build}
\end{DoxyCode}


However, the library can be customized with various options. For example, the example codes and documentation generation can be enabled by setting the {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+E\+X\+A\+M\+P\+L\+ES} and {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+D\+O\+CS} options\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\$ cmake -\/DSTARNEIG\_ENABLE\_EXAMPLES=ON -\/DSTARNEIG\_ENABLE\_DOCS=ON ../}
\end{DoxyCode}


The {\itshape installation path} can be changed during the configuration phase\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\$ cmake -\/DCMAKE\_INSTALL\_PREFIX=/path/to/somewhere/ ../}
\end{DoxyCode}


\begin{DoxyRemark}{Remarks}
The library can be compiled separately from the other software components\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\$ mkdir build-\/src}
\DoxyCodeLine{\$ cd build-\/src/}
\DoxyCodeLine{\$ cmake ../src/}
\DoxyCodeLine{\$ make}
\end{DoxyCode}


It may sometimes be necessary to compile C\+U\+DA source files with a different compiler than what {\ttfamily cmake} uses by default. For example, some C\+U\+DA version do not support G\+CC compilers that are newer than G\+CC 5 release series. In that case {\ttfamily cmake} can be configured to use a G\+CC 5 release series compiler\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\$ cmake -\/DCUDA\_HOST\_COMPILER=/usr/bin/gcc-\/5 -\/DCUDA\_PROPAGATE\_HOST\_FLAGS=OFF ../}
\end{DoxyCode}

\end{DoxyRemark}

\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+D\+O\+CS}\+: Enables documentation generation ({\ttfamily O\+FF} by default).

\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+F\+U\+L\+L\+\_\+\+T\+E\+S\+TS}\+: Enables additional tests ({\ttfamily O\+FF} by default).
\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+R\+E\+F\+E\+R\+E\+N\+CE}\+: \+: Enables reference M\+PI implementations ({\ttfamily O\+FF} by default).

\item A user must initialize and update the included G\+IT submodules.
\end{DoxyItemize}
\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+D\+I\+S\+A\+B\+L\+E\+\_\+\+M\+PI}\+: Explicitly disables the M\+PI support even when the system would support it ({\ttfamily O\+FF} by default).
\begin{DoxyItemize}
\item Must be set before {\ttfamily cmake} is run for the first time.
\end{DoxyItemize}
\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+D\+I\+S\+A\+B\+L\+E\+\_\+\+C\+U\+DA}\+: Explicitly disables the C\+U\+DA support even when the system would support it ({\ttfamily O\+FF} by default).
\begin{DoxyItemize}
\item Must be set before {\ttfamily cmake} is run for the first time.
\end{DoxyItemize}
\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+D\+I\+S\+A\+B\+L\+E\+\_\+\+B\+L\+A\+CS}\+: Explicitly disables the Sca\+L\+A\+P\+A\+C\+K/\+B\+L\+A\+CS support even when the system would support it ({\ttfamily O\+FF} by default).
\begin{DoxyItemize}
\item Must be set before {\ttfamily cmake} is run for the first time.
\end{DoxyItemize}
\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+M\+E\+S\+S\+A\+G\+ES}\+: Enable basic verbose messages ({\ttfamily ON} by default).
\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+V\+E\+R\+B\+O\+SE}\+: Enable additional verbose messages ({\ttfamily O\+FF} by default).
\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+E\+V\+E\+N\+TS}\+: Enable event traces ({\ttfamily O\+FF} by default).
\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+E\+V\+E\+N\+T\+\_\+\+P\+A\+R\+S\+ER}\+: Enable event parser ({\ttfamily O\+FF} by default).

\item {\ttfamily S\+T\+A\+R\+N\+E\+I\+G\+\_\+\+E\+N\+A\+B\+L\+E\+\_\+\+P\+R\+U\+N\+I\+NG}\+: Enable task graph pruning ({\ttfamily ON} by default).

\end{DoxyItemize}

\begin{DoxyItemize}

\item {\ttfamily L\+A\+P\+A\+C\+K\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: L\+A\+P\+A\+CK library.
\item {\ttfamily H\+W\+L\+O\+C\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: Portable Hardware Locality (hwloc) library.

\item {\ttfamily M\+P\+I\+\_\+\+Fortran\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: Fortran M\+PI library.
\item {\ttfamily S\+C\+A\+L\+A\+P\+A\+C\+K\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: Sca\+L\+A\+P\+A\+CK library.
\item {\ttfamily B\+L\+A\+C\+S\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: B\+L\+A\+CS library.
\item {\ttfamily S\+T\+A\+R\+P\+U\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: Star\+PU library.
\item {\ttfamily S\+T\+A\+R\+P\+U\+\_\+\+M\+P\+I\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: Star\+P\+U-\/\+M\+PI library.
\item {\ttfamily G\+S\+L\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: G\+NU Scientific Library.
\item {\ttfamily M\+A\+G\+M\+A\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: M\+A\+G\+MA library.
\item {\ttfamily M\+I\+S\+C\+\_\+\+L\+I\+B\+R\+A\+R\+I\+ES}\+: Miscellaneous libraries.
